# 下一步上下文读取协议

这个协议用于约束 AI 在回答项目进度类问题前的上下文读取动作，避免未检查仓库状态就凭记忆回答。

## 触发场景

当用户询问以下问题或同义表达时，先执行本协议：

- 下一步是什么
- 现在做到哪
- 继续做什么
- 当前进度如何
- 接下来应该做什么
- 阶段文件里写的下一步是什么

## 必读来源

回答前必须读取或检查以下来源：

1. `AGENTS.md`
2. `docs/plans`
3. `docs/superpowers/specs`
4. 当前分支和工作树状态
5. 最近提交记录

优先使用 `rg --files`、`rg`、`sed`、`git status`、`git branch` 和 `git log` 获取信息。

## Git 检查

至少确认：

- `git status --short`
- `git branch --show-current`
- `git log --oneline -5`

如果需要判断是否可合并或是否已推送，再补充 `git branch -vv` 或 `git rev-list --left-right --count`。

## 回答要求

回答“下一步”时必须区分：

1. 当前工作树状态。
2. 当前分支和最近提交状态。
3. 阶段文档里的依据。
4. 已完成或未完成的验收。
5. 推荐的下一步动作。

如果阶段文档和代码实际状态不一致，应明确指出不一致，并优先给出可验证的下一步。

## 禁止行为

- 禁止在未读取项目阶段文档和 Git 状态时回答“没有上下文”。
- 禁止只根据聊天记忆判断下一步。
- 禁止把阶段文档里的旧计划当作当前事实，必须结合代码、测试或 Git 状态确认。
